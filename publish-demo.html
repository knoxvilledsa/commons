About this prototype / Publishing

How updates work

Publish from a form

The fear with any new site is the same one every time: who will keep it updated? The answer here is anyone in the chapter, because publishing is a form, not code. If you can send an email, you can post the news.

Try the live demo Why a new site

The editing model, in plain language

A volunteer signs in with a Google (Gmail) account, the same login they already use every day. They open News, click New post, and a simple form appears.

The form asks for exactly three things: a Title, a Body, and an optional Image. They type the headline, write a sentence or two, drop in a photo if they have one, and click Publish.

Behind the scenes a rebuild runs automatically, and the post is live on the site in about a minute. No file to upload, no plugin to update, no server to log into.

The most important part is what the form does not show. It exposes only the content fields, never the layout, the menus, or the code. An editor cannot rearrange the page, break the design, or take the site down by accident. The worst thing that can happen is a typo, and that is a one-click fix.

Keeping the site current

Every stale page on the current site comes back to the same problem: updating it is a chore that only one person knows how to do. When that person is busy, the events page sits frozen and the news goes quiet.

This model removes that bottleneck. Posting is so easy that anyone in a working group can do it from their phone after a meeting, so the chapter never depends on a single webmaster. Upkeep is near zero, and the site stays as alive as the chapter is.
