About this prototype

About this prototype

This is a working preview of a new Knoxville Area DSA website, built front-end only to show the chapter what a rebuild could look like, feel like, and cost to run.

What this is

Every page in this prototype is a real, clickable HTML file built in the chapter’s house style (the “Knoxville Bulletin”). You can navigate the full site, toggle dark mode, and interact with forms. Nothing sends data anywhere: the forms show a success state because app.js intercepts them for the demo. The ribbon at the top of each page links back here.

The prototype was built to give the chapter something concrete to review before committing to a production rebuild. It is a preview, kept out of every search index on purpose, and shared by direct link for chapter review.

What is real

  • The chapter name, mission, and legal status: Knoxville Area DSA, a 501(c)(4) nonprofit, EIN 83-2598312
  • The General Meeting schedule: the fourth Monday of each month, hybrid, about 6:00 PM; RSVP via Action Network; next up July 27, 2026
  • The dates are live, not typed in: every recurring meeting and event on this prototype works out its own next occurrence, so the calendar was correct the day you opened it and stays correct months from now with nobody touching it. That is the same automation the real site would run, which means no volunteer has to re-enter the schedule every month
  • Working groups and their real public contacts: each reached through its interest form or the Contact page rather than a personal contact
  • Live links to national DSA join, Action Network, Instagram (@knoxvilledsa), and Facebook
  • Sibling chapters: Chattanooga DSA and Northeast TN DSA
  • The mission line, quoted verbatim: “We believe that both the economy and society should be run democratically to meet human needs, not to make profits for a few.”
  • The donate disclosure: contributions to Knoxville Area DSA are not tax-deductible

What is sample

Content that cannot be mistaken for a real chapter commitment is marked with a “Sample” label or a prototype note wherever it appears on the page that carries it. This includes:

  • Example events and their details
  • News posts and article body copy
  • Member quotes
  • Campaign numbers and progress figures

If you see content on another page and it is not marked, it should be real chapter data. If something looks wrong, let us know.

The plan after this

If the chapter approves a rebuild, the production site runs on the locked stack: Astro for the front end, Sanity on its free tier as a structured content store, and Cloudflare Pages for hosting. That comes to about 0 USD per month, plus the annual domain renewal. Editors sign in with a Gmail account, fill in a structured form, and hit Publish. A rebuild runs automatically and the change is live in about a minute. There is no server to patch, no WordPress database to back up, and no plugin that can break after an update.

The stack is approved and the webfont question is settled: DSA national permits serving Manifold as a public webfont, so the house face ships in the built pages, with Archivo kept only as a graceful fallback. A handful of decisions only the chapter can make still gate a production build:

  • A private code repository. Cloudflare Pages builds the site from a GitHub repository. The chapter needs to create a separate, private repository for the site source, kept private so the licensed font ships inside the built pages rather than as raw files.
  • Donation processor and the legal gate. The chapter needs to pick a payment path (for example ActBlue Civics, PayPal, or Stripe) and, because it is a 501(c)(4), get the Tennessee charitable solicitation exemption on file before the donate page asks for money. The donate page in this prototype is a non-functional mockup of the proposed flow.
  • Action Network access. The chapter needs to confirm its Action Network credentials so the Join form, the RSVP forms, and the welcome email sequence wire into the tools it already uses.
  • Domain control and cutover. The chapter needs to confirm who controls DNS for knoxvilledsa.org so it can plan the switch, keeping the current site reachable until the new one is live.

Most are quick confirmations; the donation legal step and the domain cutover need a little lead time. See Why a new site for the full technical and organizational case.
